# AGENTS.md

## Commands
- **Setup**: `composer run setup` (install deps, copy .env.example, generate APP_KEY, migrate, npm install+ build)
- **Dev**: `composer run dev` (artisan serve + queue:listen + vite concurrently)
- **Test all**: `composer run test` (clears config first, then `php artisan test`)
- **Single test**: `php artisan test --filter=TestName`
- **Single test file**: `php artisan test tests/Feature/ExampleTest.php`
- **Lint**: `vendor/bin/pint` (Laravel Pint, PSR-12-ish)
- **Build frontend**: `npm run build`
- **Watch frontend**: `npm run dev`

## Critical setup gotchas
- `JWT_SECRET` is **not** in `.env.example` and is **not** generated by `composer run setup`. It must be added manually to `.env` or all JWT auth will fail.
- `.npmrc` sets `ignore-scripts=true` — npm post-install scripts don't run. This matches the `npm install --ignore-scripts` in `composer.json` setup.

## Architecture

### Service-oriented routing
- All named API routes are defined in `config/service.php` and auto-registered in `routes/api.php:9-22`. Do NOT manually add routes to `routes/api.php` for standard service endpoints — add entries to `config/service.php` instead.
- `config/service.php` entries have: `name`, `class`, `type` (http method), `end_point`, `guard` (`null` = no auth; `'api'`/`'doctor'`/`'patient'`).
- The `AppServiceProvider` binds short service names to classes (`config/service.php` -> `app/Providers/AppServiceProvider.php:14-18`).

### Service pattern
- Every service extends `App\CoreService\CoreService` and implements `prepare($input)` + `process($input, $originalData)`.
- Input is auto-validated via the `validation()` method (returns Laravel rules array). If `validation()` returns empty (default), validation is skipped.
- Throw `App\CoreService\CoreException` for errors — constructor: `(message, statusCode=422, errorList=[])`.
- Services with `$transaction = true` are wrapped in DB transactions automatically (`CallService::call`).

### CallService entry points
- `CallService::run(name, input)` — used for HTTP route handlers, catches all exceptions and returns JSON via `CoreResponse`.
- `CallService::execute(name, input)` — same as `run` but also checks permission via `$object->task` using the `hasPermission()` helper.
- `CallService::call(name, input)` — raw service execution with optional transaction, throws on error (no JSON wrapping). Used internally between services.

### CRUD pattern
- Generic CRUD routes at `routes/api.php:24-31` (all behind `setguard:api` + `auth.rest`):
  - `GET /{model}/list` — paginated list
  - `GET /{model}/dataset` — same backend as list (both delegate to `Crud\Get`)
  - `GET /{model}/{id}/show` — single record (`Crud\Find`)
  - `POST /{model}/create` — insert (`Crud\Add`)
  - `PUT /{model}/update` — update by id (`Crud\Edit`)
  - `DELETE /{model}/delete` — delete by id (`Crud\Delete`)
- `CrudController` calls CRUD services by **FQCN string** (e.g. `'App\Services\Crud\Get'`), resolved via Laravel's container autowiring — they are NOT registered in `config/service.php` or `AppServiceProvider`.
- Model names are resolved via `Str::studly()` to `App\Models\{Name}`.
- Permission checks use the format:
  - `view-{model}` (list/dataset)
  - `show-{model}` (show)
  - `create-{model}` (create)
  - `update-{model}` (update)
  - `delete-{model}` (delete)
- CRUD services use `DB::table()` for all queries (NOT Eloquent). Timestamps (`created_at`, `updated_at`) are set manually. Eloquent model objects are only retrieved via `::find($id)` for passing to lifecycle hooks.
- Only fields declared in `FIELD_ADD`/`FIELD_EDIT` are persisted — the service whitelists input against these constants.

### Model conventions
- Models define schema via class constants: `FIELD_LIST`, `FIELD_ADD`, `FIELD_EDIT`, `FIELD_VIEW`, `FIELD_VALIDATION`, `FIELD_FILTERABLE`, `FIELD_SEARCHABLE`, `FIELD_SORTABLE`, `FIELD_RELATION`, `FIELD_UNIQUE`, `FIELD_UPLOAD`, `FIELD_DEFAULT_VALUE`, `FIELD_TYPE`, `FIELD_READONLY`, `FIELD_ARRAY`, `CHILD_TABLE`, `CUSTOM_SELECT`, `CUSTOM_LIST_FILTER`, `PARENT_CHILD`, `TABLE`, `FILEROOT`, `IS_ADD`, `IS_EDIT`, `IS_LIST`, `IS_DELETE`.
- `BaseModel` defines static lifecycle hooks: `beforeInsert`, `afterInsert`, `beforeUpdate`, `afterUpdate`, `beforeDelete`, `afterDelete`, `beforeList`, `afterDetil`. Always use `static` methods and match the exact signatures from `BaseModel`.
- Authenticatable models (`Users`, `Doctors`, `Patients`) extend `Authenticatable` (not `BaseModel`) but follow the same constant/hook conventions and implement `JWTSubject`. They must use `Hash::make()` for passwords — do NOT use `'hashed'` casts.
- `Doctors` and `Patients` share a `NormalizesAuthFields` trait for `normalizePasswordOnUpdate()` and `normalizeBooleanField()`.

### Auth guards
- Three JWT guards: `api` (Users/employees), `doctor` (Doctors), `patient` (Patients).
- `SetGuard` middleware (alias `setguard`) switches `auth.defaults.guard` before `AuthApiMiddleware` validates the JWT. Both are aliased in `bootstrap/app.php`.
- Login endpoints use `guard: null` in `config/service.php` — they return a JWT, don't require one.
- `hasPermission()` helper (`app/helpers/function.php`) checks `role_task` join table; role_id=1 is super-admin (always returns true).

### File uploads
- Temp uploads go to `storage/app/documents/temp/` via `UploadController::upload` (`POST /upload-tmp`).
- CRUD services use `HandlesFileUploads` trait to move temp files to `storage/app/documents/uploads/{table}/` during create/update.
- Final file paths are served at `/file/{path}` (`routes/api.php:64-69`) using the `document` disk.

### Testing
- Tests use SQLite in-memory (`phpunit.xml` sets `DB_CONNECTION=sqlite`, `DB_DATABASE=:memory:`).
- `phpunit.xml` overrides env vars: broadcasts=null, cache=array, mail=array, queue=sync, session=array.
- Migrations run automatically (Laravel's default behavior with in-memory SQLite).

### Environment quirks
- `DB_CONNECTION` in `.env` is `mysql`, tests override to `sqlite`.
- `QUEUE_CONNECTION` is `database` — queue worker must run separately in production.
- `SESSION_DRIVER` is `database` — ensure migrations are run before using the app.
- `package.json` uses `"type": "module"` (ESM).

### Key dependencies
- **tymon/jwt-auth** — JWT authentication. Sanctum is present but only for token abilities, not auth.
- **Tailwind CSS 4** via Vite plugin, with `Instrument Sans` font from Bunny Fonts.
- **Laravel Pint** for code style (no other linter).
